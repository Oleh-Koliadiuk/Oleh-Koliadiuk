About JavaScript: Architectural Concept & Engine Logic

1. The System & Runtime
- Paradigm: Dynamic, high-level, single-threaded but non-blocking (Event Loop).
- Runtime: Executing via Node.js (V8 Engine). It compiles JS code directly into
  machine code on the fly for maximum speed.

2. Global VS Code Workflow Optimization
- Language Switch: Code written purely in English to avoid hitting the 'Fn' 
  key for layout switching. Maximizes coding speed and mechanical consistency.

3. JavaScript vs Python: Paradigm Shift
- Architecture: JS is built entirely around braces {} and explicit instructions; 
  Python is built around semantic indentation (tabs/spaces).
- Strictness: JS requires variable definitions ('let' / 'const') and handles
  statements with semi-colons (;). Python creates variables implicitly.
- Output: 'console.log()' in JS vs 'print()' in Python.

4. The Core Truth About Memory (Critical)
- Primitive Types (Numbers, Strings, Booleans):
  * Stored and copied by value.
  * Changing a copy leaves the original untouched.

- Reference Types (Objects, Arrays):
  * Stored and copied by reference (shortcuts/memory addresses).
  * Variable does not hold data; it holds an address to a "box" in RAM.
  * Two empty boxes are never equal: {} == {} returns false because they
    are located at different memory addresses.

5. Cloning Strategies (Manipulating Boxes)
- Shallow Copy ({ ...user } / Object.assign):
  * Only creates a new box for the first level of properties.
  * Nested objects inside are still copied by reference! (Dangerous trap).
- Deep Copy (structuredClone(user)):
  * Recursively duplicates every single layer and sub-box in memory.
  * Complete data autonomy.
